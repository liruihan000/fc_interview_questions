# 问题7：大模型成本失控，如何优化？如何为AI产品设计商业模式？

> **原题**：大模型成本失控，你如何优化？如何为AI产品设计商业模式？

## 成本优化

在Q2的SEC系统设计中已算过这笔账——所有5000条filing用顶级模型处理日成本$50-100，结构化优先+模型路由优化到$0.3-1/天。核心：**让正确的模型做正确的事**。架构层路由决策的影响比工程层优化大一个数量级。

Livins AI折扣提取也有类似经历——最初所有listing用同一个模型，加了info_hash幂等+小模型分流后成本降一个数量级。

### 成本为什么会失控

通常不是API单价贵，而是三个结构性问题：

1. **模型选择粗放**：所有请求走同一个Premium模型，70%简单任务被过度服务
2. **架构复杂度失控**：多Agent Token消耗是单轮对话的15-20倍，多数场景单Agent够用
3. **重复推理浪费**：大量语义相似查询每次重新推理

### 优化策略（按优先级）

**模型路由（节省60-80%，最优先）。** 按请求复杂度分流：

```
用户请求 → 复杂度评估 → 简单(70%) → MiMo-V2-Flash        $0.10/M tokens
                      → 中等(20%) → MiniMax M2.5          $0.30/M tokens
                      → 复杂(10%) → MiMo-V2-Pro           $1.00/M tokens
```

以MiMo/MiniMax为例，DeepSeek V3.2、Gemini Flash等也有类似低成本选项。

**语义缓存（节省40-70%）。** 向量相似度匹配，语义相似查询直接返回缓存结果。适合FAQ、知识库问答等重复率高的场景。

**Prompt工程优化（节省20-40%）。** 精简上下文、压缩Prompt、启用Prompt Caching和Batch API。各家Provider都支持。

**推理层优化（节省30-50%，自部署场景）。** 量化、知识蒸馏、Speculative Decoding，适用于自部署开源模型。

几层不互斥，成熟系统叠加使用。优先做模型路由，决定成本量级。

## 商业模式设计

AI产品和传统SaaS最大区别：**边际成本不再趋近于零**。传统SaaS毛利率80-90%，AI产品仅50-60%——每次推理都有真实成本。

### 定价模式

| 模式 | 机制 | 适用场景 |
|------|------|----------|
| **混合订阅制** | 基础订阅 + 用量超额计费 | 大多数AI SaaS（2026主流） |
| **按量计费** | 按token/API调用/任务数 | 开发者工具、API服务 |
| **按效果计费** | 按完成任务数/节省时间 | 可量化交付的场景 |

行业共识偏向混合订阅制。但对FC这类B2B场景，按效果计费更贴切——SEC筛选系统的价值不按token算，按找到的有效客户线索数量算。每筛出一家有效中国背景公司并提取联系方式，对商务团队都是实际线索价值。

**模型路由不只是技术决策，也是商业决策。** 90%请求走低成本模型，直接决定毛利率能否从50%提到70%以上。

**重度用户的成本天花板必须设。** per-user成本上限、阶梯定价不是可选项。

定价策略和成本优化不是两个独立问题——模型路由同时决定性能体验和毛利率。
